[Java] JPA 데이터베이스 스키마 자동생성

Posted on 2021-07-11 Edited on 2021-07-11 In Java Disqus:
DB 스키마를 미리 지정하는 방법에 대해 살펴본다 🤡


✨ Contents:

  1. 개요
  2. hibernate.hbm2ddl.auto
  3. Reference

개요

  • DDL 을 애플리케이션 실행 시점에 자동으로 생성
  • DDL 은 반드시 개발 장비에서만 사용
    • 운영 서버에는 절대 사용하지 않고 다듬은 후 운영 서버에 반영하는 것이 최선

hibernate.hbm2ddl.auto 옵션

  • create: 기존테이블 삭제 후 다시 생성 (drop → create)
  • create-drop: create 와 동일하지만 애플리케이션 종료 이후 drop
  • update: 변경 부분만 반영
  • validate: 엔티티 <-> 테이블 매핑만 확인
  • none: 사용하지 않음
    • 운영 장비에는 create & create-drop & update 절대 사용하지 않도록 주의
    • 개발 초기: create or update
    • 테스트 서버: update or validate
    • 스테이징 및 운영 서버: validate or none
  • @Column annotation 으로 제약 조건 추가가 가능
    • 단, DB 설정을 변경하는 것이지 JPA 실행 로직에는 전혀 영향이 없음
