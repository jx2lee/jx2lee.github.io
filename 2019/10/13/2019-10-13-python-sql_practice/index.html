Python - N진수의 각 자리 숫자 곱의 최댓값 구하기

문제

문제 설명
어떤 수의 0이 아닌 각 자릿수를 모두 곱한 값을 자릿수 곱이라고 합니다. 자릿수 곱은 주어진 수를 몇 진수로 나타내느냐에 따라 달라질 수가 있습니다. 예를 들어, 10진수 10을 3진수로 나타내면 101이며, 자릿수 곱은 1입니다. 그리고 4진수로 나타내면 22이며, 자릿수 곱은 4입니다.
10진수 N을 k(1 < k < 10) 진법으로 나타내려고 할 때, 자릿수 곱이 최대가 되는 k와 그때의 자릿수 곱을 구해주세요. 만약 자릿수 곱이 최대가 되는 k가 여러 개 있다면 그중 가장 큰 k를 구해주세요.
예를 들어 N = 10일 때, 4진수로 나타내면 22이고, 0이 아닌 자릿수들의 곱은 4입니다. 또한, 6진수로 나타내면 14이고, 0이 아닌 자릿수들의 곱은 4입니다. 두 경우 모두 진법 자릿수 곱이 최대가 되는데 둘 중 k가 더 큰 경우는 6진수로 나타낼 때입니다.
10진수 N이 매개변수로 주어질 때, 자릿수 곱이 최대가 되는 k 진법과 그때의 자릿수 곱을 return 하는 solution 함수를 완성해주세요.

제한사항

  • N은 10 이상 1,000,000 이하의 정수입니다.
  • return하는 배열은 [자릿수 곱이 최대가 되는 k 진법, 그때의 자릿수 곱] 순서여야 합니다.

입출력 예

N result
10 [6,4]
14 [5,8]

입출력 예 설명
입출력 예 #1문제의 예시와 같습니다.

입출력 예 #2
10진수 14를 5진수로 나타내면 24이고, 0이 아닌 자릿수들의 곱은 8이며, 이때가 최대입니다.

접근

우선은 함수 두 개로 접근하였다. N진법으로 표현하는 함수, 다른 하나는 문제의 답을 구하는 solution 함수이다.

N진법으로 표현하는 함수(get_n)*는 divmod내장함수를 이용해 몫과 나머지를 구해나간다. 이후 나머지(q)*와 t를 이용해 N진법으로 표현한 수를 구한다.

  • 변수 설명
    • n, base : N진법으로 표현하고자 하는 수(십 진수)와 표현하고자 하는 진수(N)
    • q, r : 몫, 나머지

solution 함수는 다음과 같이 구성된다. answer 변수에 주어진 수에 대한 N진법의 각 숫자 곱을 구한다. 입력값에 대한 N진법은 tmp에 담아두며 0이 아닌 수를 곱해 answer에 담고 (ans는 각 자리 숫자의 곱), 이를 key 기준 정렬 후 결과값을 return 한다.

문제 설명

|TITLE|VARCHAR(N)|FALSE|

|TAGS|JSON|FALSE|
|VIEWS|INTEGER|FALSE|
|LIKES|INTEGER|FALSE|
|DISLIKES|INTEGER|FALSE|

SQL - 친구가 없는 유저에게 선물을 !

문제

문제 설명

GAME_USERS 테이블은 XX런 게임 유저의 정보를 담고 있는 테이블입니다. GAME_USERS 테이블 구조는 다음과 같으며, ID, DISTANCE, TIME_SPENT, BEST_DATE는 각각 유저의 아이디, 최고 기록(미터 단위), 최고 기록 달성시 플레이한 시간, 최고기록 경신 날짜를 나타냅니다.

NAME TYPE
ID VARCHAR
DISTANCE INT
TIME_SPENT DECIMAL
BEST_DATE DATETIME

FRIENDS 테이블은 게임 내 친구 관계 정보를 담고 있습니다. FRIENDS 테이블 구조는 다음과 같으며 ID1, ID2 쌍은 ID1과 ID2가 서로 친구라는 뜻입니다. 이 테이블에서, 친구 관계는 양방향이며, 친구 관계가 중복으로 등록되는 경우는 없습니다. (id1가 id2의 친구이면 id2는 id1의 친구이고, 레코드 id1, id2가 등록되어 있을 때, 레코드 id2, id1가 등록되는 경우는 없습니다)

NAME TYPE NULLABLE
ID1 VARCHAR(N) FALSE
ID2 VARCHAR(N) FALSE

XX런 게임 유저 중, 친구가 한 명도 없는 유저를 대상으로 이벤트를 진행하려 합니다. 친구가 한 명도 없는 유저의 아이디를 사전 순으로 조회하는 SQL 문을 작성해주세요.

예시

예를 들어 GAME_USERS 테이블과 FRIENDS 테이블이 다음과 같다면

GAME_USERS:

ID DISTANCE TIME_SPENT BEST_DATE
mahjongsooty678 59400 4.721477229 2016-11-02 20:48:41
ayrmogul500 99000 4.376248407 2016-12-09 14:18:32
crestrumble1879 118300 3.033516385 2016-11-11 09:05:23
dineronoodle1236 59500 4.987321227 2016-12-22 02:26:06
formflat1915 15200 2.954442763 2016-11-03 23:35:53

FRIENDS:

ID1 ID2
ayrmogul500 frenchchampagne1501
crestrumble1879 toucanmoaning1646
chamaeleonbullhorn1467 crestrumble1879
mastmilo1852 formflat1915
  • mahjongsooty678는 친구가 없습니다.
  • ayrmogul500는 친구가 한 명 있습니다.
  • crestrumble1879는 친구가 두 명 있습니다.
  • dineronoodle1236는 친구가 없습니다.
  • formflat1915는 친구가 한 명 있습니다.

따라서 SQL 문을 실행하면 다음과 같이 나와야 합니다.

USER_ID
dineronoodle1236
mahjongsooty678

접근

핵심은 friends 테이블을 이해하는 것이었다.

처음에는 friends 테이블을 잘못 이해하여, id가 컬럼 id1 또는 컬럼 id2에 포함하지 않아도 되는 쿼리를 작성하였다. 하지만, 테이블을 자세히 살펴보면 문제는 친구가 없는 유저를 구해야하므로 아무런 친구관계가 없어야 한다. 이는 곧 id1, id2에 모두 속하지 않아야 한다.

따라서, where 절 부분을 and를 이용해 friends 내 칼럼 id1, id2에 모두 포함되지 않는 id를 조회하면 된다. 위 점만 주의하면 문제를 쉽게 풀 수 있을 것이다.
